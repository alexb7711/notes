#+title: Fuzzy Theory

* Introduction
:PROPERTIES:
:CUSTOM_ID: introduction
:END:
Fuzzy logic is a form of artificial intelligence that resembles the
human decision-making methodology where we allow vague and imprecise
information. In other words, it allows for imprecise (non-binary)
decision to be modeled #3. Contradictory to one's
idea of fuzzy logic, fuzzy logic is not logic that is fuzzy, but logic
that is is used to describe fuzziness #1.

[[https://www.tutorialspoint.com/fuzzy_logic/images/fuzzy_logic_introduction.jpg]]

An important fact about fuzzy logic is that it does not require a
model/. It works well with complex system where the mechanism are not
fully known. As long as you have some intuition about the system, then
the rules can be developed in implemented #3.

The applied rules are the encoding of knowledge. For example, assume a
banker wants to figure out the risk of giving a person a loan. We can
say

- Good credit \(\rightarrow\) low risk
- Neutral credit \(\rightarrow\) medium risk
- Bad credit \(\rightarrow\) high risk

The set of rules above are the /encoding of knowledge/ that can be used
for predictions #3.

** Fuzzy Set
:PROPERTIES:
:CUSTOM_ID: fuzzy-set
:END:
Fuzzy [[file:set_theory.org][sets]] allows elements to have varying
degrees of memberships in sets. For example consider the image below

[[https://www.tutorialspoint.com/fuzzy_logic/images/fuzzy_classical_set.jpg]]

The membership function of classical sets \(A\) is binary in the sense
that the element \(a_i\) is either in the set or not. On the other hand,
the membership function of fuzzy set \(\tilde{A}\) has elements
\(\tilde{a}_i\) that have /varying degrees of membership in the set/. In
other words, the membership functions describe the degree of truth in
fuzzy logic.

*** Operations of Fuzzy Sets
:PROPERTIES:
:CUSTOM_ID: operations-of-fuzzy-sets
:END:
- Union/Fuzzy 'OR'
  - \(\mu _{{\widetilde{A}\cup \widetilde{B} }}\left ( y \right ) = \mu _{\widetilde{A}}\vee \mu _\widetilde{B} \quad \forall y \in U\)
    where \(\vee\) is the \(max\) operation.
  - [[https://www.tutorialspoint.com/fuzzy_logic/images/union.jpg]]
- Intersection/Fuzzy 'AND'
  - \(\mu _{{\widetilde{A}\cap \widetilde{B} }}\left ( y \right ) = \mu _{\widetilde{A}}\wedge \mu _\widetilde{B} \quad \forall y \in U\)
    where \(\wedge\) represents the \(min\) oepration
  - [[https://www.tutorialspoint.com/fuzzy_logic/images/intersection.jpg]]
- Complement/Fuzzy 'NOT”
  - \(\mu _{\widetilde{A}} = 1-\mu _{\widetilde{A}}\left ( y \right )\quad y \in U\)
  - [[https://www.tutorialspoint.com/fuzzy_logic/images/complement.jpg]]

*** Operations of Trapozoidal/Triangular Fuzzy Sets
:PROPERTIES:
:CUSTOM_ID: operations-of-trapozoidaltriangular-fuzzy-sets
:END:
\begin{equation}
\begin{array}{l}
    \text{let } \tilde{A}_1 = (a_1, b_1, c_1, d_1) \\
    \text{let } \tilde{A}_2 = (a_2, b_2, c_2, d_2) \\
    \tilde{A}_1+\tilde{A}_2 = (a_1+a_2, b_1+b_2, c_1+c_2, d_1+d_2) \\
    \tilde{A}_1-\tilde{A}_2 = (a_1-d_2, b_1-c_2, c_1-b_2, d_1-a_2) \\
    \tilde{A}_1*\tilde{A}_2 = (a_1 a_2, b_1 b_2, c_1 c_2, d_1 d_2) \\
\end{array}
\end{equation}

** Membership Functions
:PROPERTIES:
:CUSTOM_ID: membership-functions
:END:
Mathematically, membership functions are denoted as
#1

\[
\widetilde{A} = \left \{ \left ( y,\mu _{\widetilde{A}} \left ( y \right ) \right ) | y\in U\right \}
\]

*** Membership Function Properties
:PROPERTIES:
:CUSTOM_ID: membership-function-properties
:END:
- Core: The set of values \(y\) such that \(\mu_{\tilde{A}}(y) = 1\)
- Support: The set of values \(y\) such that \(\mu_{\tilde{A}}(y) > 0\)
- Boundary: \(1 > \mu_{\tilde{A}}(y) > 0\)

*** Fuzzification
:PROPERTIES:
:CUSTOM_ID: fuzzification
:END:
This is the process of transforming crisp sets into fuzzy sets. This
process is used to determine how much the crisp number lies in each
given region. In the example of determining risk for giving a loan in
#2, the example of a crisp value being fuzzified
was \(660 = ![pic](0,0.9,0.1)\) where the fuzzy value is bad, neutral,
and good credit, respectively. There are multiple methods of doing this,
two of methods are shown below #1

[[https://www.researchgate.net/profile/Abderrahim_Bennassar/publication/286013789/figure/fig2/AS:307781908353025@1450392333054/Block-diagram-of-a-fuzzy-logic-controller.png]]

**** Support Fuzzification (s-fuzzification) Method
:PROPERTIES:
:CUSTOM_ID: support-fuzzification-s-fuzzification-method
:END:
\[
\widetilde{A} = \mu _1Q\left ( x_1 \right )+\mu _2Q\left ( x_2 \right )+...+\mu _nQ\left ( x_n \right )
\]

where \(Q(x_i)\) is called the /kernel of fuzzification/. This method is
implemented by keeping \(\mu_i\) constant and \(x_i\) is transformed
into a fuzzy set \(Q(x_i)\).

**** Grade Fuzzification (g-fuzzification) Method
:PROPERTIES:
:CUSTOM_ID: grade-fuzzification-g-fuzzification-method
:END:
This is very similar to the s-fuzzification, but \(x_i\) is held
constant and \(\mu_i\) is expressed as a fuzzy set.

*** Defuzzification
:PROPERTIES:
:CUSTOM_ID: defuzzification
:END:
This is the process of transforming fuzzy sets into crisp sets
#1. This process happens after inferences have been
made on the fuzzy numbers. This usually implies another set of fuzzy
rules that determines the ranges of memberships #2.
Similar to the fuzzification process, there are multiple ways to
defuzzify values #1:

{{https://www.researchgate.net/profile/Abderrahim_Bennassar/publication/286013789/figure/fig2/AS:307781908353025@1450392333054/Block-diagram-of-a-fuzzy-logic-controller.png|Fuzzy
logic block diagram}}

**** Max-Membership Method
:PROPERTIES:
:CUSTOM_ID: max-membership-method
:END:
This method is limited to peak output functions. Mathematically it is
represented as

\[
\mu _{\widetilde{A}}\left ( x^* \right )>\mu _{\widetilde{A}}\left ( x \right ) \: for \:all\:x \in X
\]

where \(x^*\) is the defuzzified output.

**** Centroid Method
:PROPERTIES:
:CUSTOM_ID: centroid-method
:END:
Also known as the center of area or the center of gravity method:

\[
x^* = \frac{\int \mu _{\widetilde{A}}\left ( x \right ).xdx}{\int \mu _{\widetilde{A}}\left ( x \right ).dx}
\]

What this means is that given the fuzzy value to be defizzified, draw a
horizontal line at the percentage of the membership as indicated by the
fuzzy value, merge the areas together, and find the centroid of the
geometric shape that was generated #2.

**** Weighted Average Method
:PROPERTIES:
:CUSTOM_ID: weighted-average-method
:END:
Each membership function is weighted by its maximum membership value:

\[
x^* = \frac{\sum \mu _{\widetilde{A}}\left ( \overline{x_i} \right ).\overline{x_i}}{\sum \mu _{\widetilde{A}}\left ( \overline{x_i} \right )}
\]

**** Mean-Max Membership
:PROPERTIES:
:CUSTOM_ID: mean-max-membership
:END:
Also known as the middle of the maxima

\[
x^* = \frac{\displaystyle \sum_{i=1}^{n}\overline{x_i}}{n}
\]

*** Fuzzy Arithmetic
:PROPERTIES:
:CUSTOM_ID: fuzzy-arithmetic
:END:
If we have a nonnegative, triangular fuzzy number
\(\tilde{a} = (a_1, a_2, a_3)\) and \(\tilde{b} = (b_1, b_2, b_3)\),
then the sum is defined as

\[
\tilde{a}+\tilde{b} = (a_1 + b_1, a_2 + b_2, a_3 + b_3)
\]

and the difference is

\[
\tilde{a}+\tilde{b} = (a_1 - b_3, a_2 - b_2, a_3 - b_1)
\]

** Comparing Fuzzy Numbers
:PROPERTIES:
:CUSTOM_ID: comparing-fuzzy-numbers
:END:
This is the same concept as defuzzification,
but now we will put in context of its use. It is important to be able to
compare fuzzy numbers, especially for [[file:fuzzy_lin_prog.org][Fuzzy Linear Programming]] so one is able to state whether
\(\tilde{a} < \tilde{b}\) or \(\tilde{a} > \tilde{b}\) when defining
constraints. These methods are known as _ordering function. A common
ordering function is simply taking the average of the fuzzy number
#1, #3:

\[
R(\tilde{a}) = \frac{a + m + b}{3}
\]
